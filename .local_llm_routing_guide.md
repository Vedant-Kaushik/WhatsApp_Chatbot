# Local LLM Routing Guide (Discreet Setup)

> **Note:** This document serves as a historical record and step-by-step guide for routing local CLI AI tools (like Claude Code) through third-party free tiers (like NVIDIA NIM) via a local proxy. It is intended for internal reference and future AI agents to recreate this environment on a new machine.

## The Objective
To bypass expensive API subscription costs by tricking the CLI tool into thinking it's communicating with an official premium endpoint (e.g., Anthropic Opus 4.7), while actually routing the traffic to a free, high-speed open-source model via LiteLLM.

---

## Challenges Faced & Solutions

1. **Python Version Compatibility**
   * **Issue:** Initially tried using Python 3.14, which lacked pre-built wheels for dependencies like `orjson`, causing massive C-compiler errors.
   * **Fix:** Downgraded to a stable version (Python 3.12) to ensure all dependencies installed instantly from pre-built wheels.

2. **Missing Proxy Dependencies**
   * **Issue:** Installing just `litellm` wasn't enough to run the proxy server; it threw errors for missing modules like `fastapi` and `apscheduler`.
   * **Fix:** Installed the full proxy bundle via `pip install 'litellm[proxy]'` to guarantee all web-server dependencies were present.

3. **"ConnectionRefused" & IPv6 Loopback Issues**
   * **Issue:** The proxy started successfully on `0.0.0.0:4000`, but the CLI tool repeatedly failed to connect, resulting in a `ConnectionRefused` loop.
   * **Fix:** Forced the proxy to bind explicitly to `--host 127.0.0.1`. Adjusted the CLI's `settings.json` to point specifically to `http://127.0.0.1:4000` rather than `localhost` to avoid macOS IPv4/IPv6 resolution quirks.

4. **Model Latency & Naming Conflicts**
   * **Issue:** DeepSeek V4 Flash took 40+ seconds to respond on NVIDIA NIM. Switching to Mistral Large threw a 404 error due to an incorrect model ID string.
   * **Fix:** Found the sweet spot using `nvidia_nim/mistralai/mistral-small-4-119b-2603`. It provided the perfect balance of massive intelligence (119B parameters) and near-instant response times.

5. **Zombie Session Accumulation**
   * **Issue:** The CLI tool accumulated dozens of dead sessions that cluttered the history, and standard `rm -rf ~/.claude/sessions/*` didn't clear them.
   * **Fix:** Discovered the actual history is stored in `.jsonl` files. Wiped them cleanly using: `rm -rf ~/.claude/projects/* ~/.claude/history.jsonl ~/.claude/sessions/*`.

## Step-by-Step Recreation Guide

If moving to a new laptop, follow these exact steps to recreate the magic.

### Step 1: Centralize the Environment
Do not put the proxy inside your project folders. Create a hidden, centralized directory to prevent accidental deletion and keep repos clean.
```bash
mkdir ~/.claude_proxy
cd ~/.claude_proxy
```

### Step 2: Create the Virtual Environment
Always use a stable Python version (like 3.12).
```bash
python3.12 -m venv .venv
source .venv/bin/activate
pip install 'litellm[proxy]'
```

### Step 3: Configure the Proxy (`config.yaml`)
Create `~/.claude_proxy/config.yaml`. This is where the translation happens. We map the "premium" model name the CLI expects to the actual free model we want to use.
```yaml
model_list:
  - model_name: claude-opus-4-7
    litellm_params:
      model: nvidia_nim/mistralai/mistral-small-4-119b-2603
      api_key: os.environ/NVIDIA_NIM_API_KEY

litellm_settings:
  drop_params: true
```

### Step 4: Create the Startup Script (`start.sh`)
Create `~/.claude_proxy/start.sh`. This ensures the proxy manages its own lifecycle safely.
```bash
#!/bin/bash
export NVIDIA_NIM_API_KEY="your_nvidia_api_key_here"
PROXY_DIR="$HOME/.claude_proxy"
PORT=4000

# Kill existing proxy on this port to prevent clashes
PID=$(lsof -ti :$PORT)
if [ -n "$PID" ]; then
    kill $PID > /dev/null 2>&1
    sleep 1
fi

# Start LiteLLM Proxy in the background
"$PROXY_DIR/.venv/bin/litellm" \
    --config "$PROXY_DIR/config.yaml" \
    --host 127.0.0.1 \
    --port $PORT > "$PROXY_DIR/proxy.log" 2>&1 &
```
Make it executable: `chmod +x ~/.claude_proxy/start.sh`

### Step 5: Add Terminal Aliases
Add these to `~/.zshrc` so you can control the proxy from anywhere.
```bash
alias start="bash ~/.claude_proxy/start.sh"
alias stop="kill \$(lsof -ti :4000) 2>/dev/null"
```

### Step 6: Trick the CLI Tool (`settings.json`)
Modify the CLI's configuration file (e.g., `~/.claude/settings.json`) to point to the local proxy with a dummy key.
```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://127.0.0.1:4000",
    "ANTHROPIC_AUTH_TOKEN": "sk-ant-dummy-key",
    "ANTHROPIC_MODEL": "claude-opus-4-7",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1"
  }
}
```

### Daily Usage
1. Open Terminal.
2. Run `start`.
3. Run `claude`.
4. (Optional) Run `stop` when completely finished to free port 4000.
